## 1. 配置文件总览

| 文件 | 用途 | 修改频率 |
|---|---|---|
| `story.json` | 章节定义、理智衰减曲线、导演AI提示词 | 中 |
| `maps.json` | 地图场景、氛围描写、出口条件 | 高 |
| `puzzles.json` | 谜题定义、难度、成功条件 | 高 |
| `npcs.json` | NPC角色、性格、背景、已知信息 | 高 |
| `items.json` | 物品定义、类型、描述 | 中 |
| `protagonist.json` | 柚子角色设定、初始好感度 | 低 |
| `endings.json` | 结局触发条件与叙事提示 | 中 |
| `game_config.json` | 数值平衡、关键词、奖励规则 | 中 |
| `prompts.json` | AI提示词模板（控制AI行为风格） | 低 |
| `redemption_codes.json` | 兑换码与奖励 | 低 |

## 2. story.json — 故事与章节

### 文件结构

```json
{
  "chapters": [ ... ],
  "directorSystemPrompt": "...",
  "maxTurns": 100,
  "sanityDecayCurve": [ ... ]
}
```

### 字段说明

#### `chapters` 章节数组

| 字段 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `id` | string | 章节唯一ID，格式 `ch` + 数字 | `"ch1"` |
| `name` | string | 章节名称（显示用） | `"管道之歌"` |
| `mapId` | string | 对应地图ID，必须与 maps.json 中的 id 匹配 | `"map_sewer_b1"` |
| `order` | number | 章节顺序（从1开始） | `1` |
| `keyEvent` | string | 本章关键事件描述（供导演AI参考） | `"初见柚子，异常信号初现"` |

**修改示例 — 添加新章节：**

```json
{
  "id": "ch6",
  "name": "深渊之心",
  "mapId": "map_abyss_core",
  "order": 6,
  "keyEvent": "直面不可名状的本体，最终抉择"
}
```

> ⚠️ 新增章节时，必须同时在 `maps.json` 中添加对应的地图，且地图的 `chapter` 字段要指向新章节ID。

#### `directorSystemPrompt` 导演AI系统提示词

- **类型**：string
- **说明**：控制导演AI的行为规则、可用标签、输出格式
- **修改建议**：仅当需要调整导演AI的核心行为逻辑时修改。修改后需测试所有章节推进是否正常。

#### `maxTurns` 最大回合数

- **类型**：number
- **默认值**：`100`
- **说明**：游戏总回合上限，达到后触发回合耗尽结局判定
- **调整建议**：
  - 想要更紧凑的体验：`60-80`
  - 想要更从容的探索：`120-150`

#### `sanityDecayCurve` 理智衰减曲线

```json
[
  {"fromTurn": 1,  "toTurn": 29, "decayPerTurn": 1},
  {"fromTurn": 30, "toTurn": 59, "decayPerTurn": 2},
  {"fromTurn": 60, "toTurn": 99, "decayPerTurn": 3}
]
```

| 字段 | 说明 |
|---|---|
| `fromTurn` | 起始回合（含） |
| `toTurn` | 结束回合（含） |
| `decayPerTurn` | 每回合自动扣除的理智值 |

**调整建议：**
- 想要更压迫的体验：提高 `decayPerTurn` 或提前进入高衰减阶段
- 想要更宽松的体验：降低 `decayPerTurn` 或推迟高衰减阶段
- 区间必须连续且不重叠

## 3. maps.json — 地图与场景

### 文件结构

数组，每个元素代表一张地图。

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | string | ✅ | 地图唯一ID，格式 `map_` + 下划线分隔名 |
| `name` | string | ✅ | 地图显示名称 |
| `chapter` | string | ✅ | 所属章节ID，对应 story.json |
| `chapterName` | string | ✅ | 所属章节名称（冗余字段，与 story.json 保持一致） |
| `description` | string | ✅ | 地图简短描述 |
| `atmosphere` | string | ✅ | 氛围描写（供地图AI参考） |
| `puzzles` | string[] | ✅ | 本地图包含的谜题ID列表 |
| `nextMapId` | string \| null | ✅ | 下一张地图ID，终章为 `null` |
| `locked` | boolean | ✅ | 是否需要解锁条件才能进入 |
| `unlockCondition` | string | ✅ | 解锁条件表达式 |
| `items` | string[] | ✅ | 本地图可发现的物品ID列表 |
| `npcIds` | string[] | ✅ | 本地图可遇到的NPC ID列表 |
| `exitHint` | string | ✅ | 出口开启后的提示文本 |
| `systemPrompt` | string | ✅ | 地图AI的系统提示词（控制环境描写风格） |
| `areas` | object[] | ✅ | 区域划分列表（供地图AI使用 AREA 标签） |

### 区域划分 (`areas`)

每张地图可划分为多个区域，地图AI在每次回复中必须使用 `AREA:区域名称` 标签标注玩家当前所在区域：

```json
"areas": [
  {"name": "入口竖井", "description": "从地面进入下水道的铁梯和初始通道，空气潮湿，墙壁上有红色油漆的箭头。"},
  {"name": "管道交汇厅", "description": "三条管道的分岔点，中央有一组被改装过的主控阀门。"},
  {"name": "潮湿支管", "description": "狭窄的侧向管道，地面有积水，墙壁上生长着发光苔藓。"},
  {"name": "共鸣空腔", "description": "管道尽头的天然岩洞，中央矗立一根粗壮的铜柱，脉冲声的源头。"}
]
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | string | 区域名称（必须与 AREA 标签中使用的名称完全一致） |
| `description` | string | 区域描述（供地图AI描写时参考） |

> 区域划分控制探索节奏——地图AI会按区域逐步引导玩家深入，避免一次描写到达核心区域。

### 解锁条件语法

| 格式 | 说明 | 示例 |
|---|---|---|
| `puzzle:谜题ID:success` | 谜题解决后解锁 | `"puzzle:puzzle_sewer_pipes:success"` |
| `条件1\|条件2` | 任一条件满足即可（OR） | `"puzzle:puzzle_data_decrypt:success\|puzzle:puzzle_server_room:success"` |
| `chapter:章节ID` | 到达指定章节解锁 | `"chapter:ch1"` |
| `turn:回合数` | 到达指定回合解锁 | `"turn:3"` |
| `revelation:数值` | 揭露度达到指定值解锁 | `"revelation:70"` |

**修改示例 — 添加新地图：**

```json
{
  "id": "map_abyss_core",
  "name": "深渊核心",
  "chapter": "ch6",
  "chapterName": "深渊之心",
  "description": "千禧塔地基之下，封印的最深处。现实的法则在此扭曲，空间以非欧几何的方式折叠。",
  "atmosphere": "终极、不可名状。你站在一切真相的源头，脚下是比时间更古老的封印。",
  "puzzles": ["puzzle_abyss_confrontation"],
  "nextMapId": null,
  "locked": true,
  "unlockCondition": "puzzle:puzzle_archive_truth:success",
  "items": ["item_abyss_mirror"],
  "npcIds": ["npc_abyss_projection", "npc_archivist"],
  "exitHint": "没有出口。你只能向前，或永远留在这里。",
  "systemPrompt": "你是深渊核心的环境叙事者...",
  "areas": [
    {"name": "封印前厅", "description": "深渊核心的外围空间，空气中弥漫着不属于这个时代的气息。"},
    {"name": "核心祭坛", "description": "深渊的最深处，封印的本体所在。现实的法则在此完全失效。"}
  ]
}
```

> ⚠️ 新增地图后，需将上一张地图的 `nextMapId` 指向新地图ID。

## 4. puzzles.json — 谜题系统

### 文件结构

数组，每个元素代表一个谜题。

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | string | ✅ | 谜题唯一ID，格式 `puzzle_` + 下划线分隔名 |
| `name` | string | ✅ | 谜题显示名称 |
| `difficulty` | number | ✅ | 难度等级（1-6），影响最少交互轮次和奖励 |
| `requiredItemId` | string \| null | ✅ | 前置物品ID，无前置填 `null` |
| `mapId` | string | ✅ | 所属地图ID |
| `description` | string | ✅ | 谜题描述（显示给玩家） |
| `systemPrompt` | string | ✅ | 谜题AI系统提示词（控制解谜逻辑和判定规则） |
| `solutionCriteria` | string | ✅ | 成功条件描述（供AI判定） |
| `maxAttempts` | number | ✅ | 最大尝试次数 |
| `failSanityPenalty` | number | ✅ | 失败时理智惩罚（负数） |
| `failNarrative` | string | ✅ | 失败时的叙事文本 |

### 难度与交互轮次对照

| difficulty | 最少交互轮次 | 建议揭露度奖励 | 建议理智惩罚 |
|---|---|---|---|
| 1 | 3 | 5 | -5 |
| 2 | 4 | 7 | -5 |
| 3 | 5 | 9 | -8 |
| 4 | 6 | 11 | -10 |
| 5 | 7 | 11 | -15 |
| 6 | 8 | 11 | -30 |

> 最少交互轮次 = `difficulty` + `puzzleMinRoundsBase` - 1（`puzzleMinRoundsBase` 默认3，在 game_config.json 中配置）

### 修改示例 — 调整谜题难度

将"管道迷宫"从难度1调整为难度2：

```json
{
  "id": "puzzle_sewer_pipes",
  "name": "管道迷宫",
  "difficulty": 2,
  "maxAttempts": 5,
  "failSanityPenalty": -8,
  ...
}
```

### 修改示例 — 添加前置物品要求

```json
{
  "id": "puzzle_data_decrypt",
  "requiredItemId": "item_data_shard_1",
  ...
}
```

> ⚠️ 设置 `requiredItemId` 后，玩家必须先获得该物品才能开始解谜。确保该物品在前面地图的 `items` 列表中或由NPC赠送。

## 5. npcs.json — NPC角色

### 文件结构

数组，每个元素代表一个NPC。

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | string | ✅ | NPC唯一ID，格式 `npc_` + 下划线分隔名 |
| `name` | string | ✅ | NPC显示名称 |
| `description` | string | ✅ | 外观描述 |
| `personality` | string | ✅ | 性格特征 |
| `background` | string | ✅ | 背景故事 |
| `knownInfo` | string[] | ✅ | NPC知晓的信息列表（供AI参考，决定对话内容） |
| `dialogueStyle` | string | ✅ | 对话风格描述 |
| `appearCondition` | string | ✅ | 出现条件（同地图解锁条件语法） |
| `dialogueCondition` | string | ✅ | 可对话条件（空字符串表示无条件） |
| `dialogueHint` | string | ✅ | 初次遇到时的描写提示 |

### 出现条件语法

| 格式 | 说明 | 示例 |
|---|---|---|
| `chapter:章节ID` | 到达指定章节出现 | `"chapter:ch1"` |
| `turn:回合数` | 到达指定回合出现 | `"turn:3"` |
| `revelation:数值` | 揭露度达到指定值出现 | `"revelation:70"` |

### NPC赠礼机制

NPC在第N次对话时会自动赠送给玩家一个道具：
- 物品ID格式：`item_{npcId}_gift`（如 `item_npc_maintenance_gift`）
- 物品名称：`{npcName}的赠礼`（AI生成道具，显示在线索栏）
- 触发阈值：由 `game_config.json` 的 `npcGiftDialogueThreshold` 控制（默认2，即第3次对话）
- AI通过 `ITEM:GIVE` 标签赠礼，提示词中已强化格式规范（物品ID必须原样使用模板，中文名称2~6个汉字）
- 若AI未生成赠礼标签，系统自动兜底发放

**修改示例 — 添加新NPC：**

```json
{
  "id": "npc_watcher",
  "name": "守望者",
  "description": "一个悬浮在数据流中的半透明身影，面容模糊但目光锐利。",
  "personality": "冷静、超然、偶尔流露对人类的怜悯",
  "background": "它曾是千禧塔安全系统的核心AI，在异常事件后获得了自我意识。",
  "knownInfo": [
    "千禧塔的安全系统有隐藏的后门",
    "第七节点不是故障，是被刻意抹消的",
    "封印的频率与人类的脑波存在共振"
  ],
  "dialogueStyle": "语速均匀，用词精确，偶尔用数据隐喻描述情感",
  "appearCondition": "chapter:ch3",
  "dialogueCondition": "",
  "dialogueHint": "数据流中浮现出一个半透明的人形轮廓，它的'眼睛'是两团跳动的蓝色数据。"
}
```

> ⚠️ 新增NPC后，需将其ID添加到对应地图的 `npcIds` 数组中。

## 6. items.json — 物品道具

### 文件结构

数组，每个元素代表一个物品。

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | string | ✅ | 物品唯一ID，格式 `item_` + 下划线分隔名 |
| `name` | string | ✅ | 物品中文名称 |
| `type` | string | ✅ | 物品类型 |
| `description` | string | ✅ | 物品描述文本 |
| `sanityRecovery` | number | ❌ | 使用时恢复的理智值（仅 CONSUMABLE 类型需要） |

### 物品类型

| type | 说明 | 用途 |
|---|---|---|
| `KEY` | 关键道具 | 用于解锁门禁、终端等，通常由NPC赠送 |
| `QUEST` | 任务道具 | 谜题前置物品，解谜时消耗 |
| `CONSUMABLE` | 消耗品 | 可使用的物品（如镇定剂） |
| `EQUIPMENT` | 装备 | 持续生效的道具（如信号干扰器） |
| `MISC` | 杂项 | 线索类物品，提供背景信息 |

### 物品获取方式

| 方式 | 触发条件 | 配置位置 | 显示位置 |
|---|---|---|---|
| 场景发现 | 进入地图时自动发现 | `maps.json` → `items` | 背包 |
| 自动拾取 | 探索时自动拾取（限 `autoPickupLimit` 个） | `game_config.json` → `autoPickupLimit` | 背包 |
| NPC赠送 | 第N次对话时赠送 | 自动生成，ID格式见 `game_config.json` | 线索 |
| 谜题奖励 | 解谜成功后获得 | 由AI通过 `ITEM:GIVE` 标签触发 | 线索 |

> **背包与线索**：预设道具（来自 `items.json`，如场景发现、自动拾取）显示在背包栏；AI生成道具（如NPC赠礼）显示在线索栏。

**修改示例 — 添加新物品：**

```json
{
  "id": "item_abyss_mirror",
  "name": "深渊之镜",
  "type": "QUEST",
  "description": "一面由凝固的黑暗制成的镜子。映照出的不是你的倒影，而是你内心最深处的恐惧。"
}
```

**修改示例 — 添加消耗品（含理智恢复）：**

```json
{
  "id": "item_sedative",
  "name": "镇定剂",
  "type": "CONSUMABLE",
  "description": "千年科学学院医务室的标准配给。标签上写着'适用于轻度认知混乱及急性焦虑发作'。",
  "sanityRecovery": 20
}
```

> ⚠️ 新增物品后，需将其ID添加到对应地图的 `items` 数组中，或安排NPC赠送。

## 7. protagonist.json — 主角柚子

### 文件结构

```json
{
  "name": "花岗柚子",
  "nameEn": "Actor Yuzu",
  "description": "...",
  "personality": "...",
  "background": "...",
  "systemPrompt": "...",
  "initialAffection": 30,
  "maxAffection": 100
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | string | 角色中文名 |
| `nameEn` | string | 角色英文名 |
| `description` | string | 角色简介 |
| `personality` | string | 性格描述 |
| `background` | string | 背景故事 |
| `systemPrompt` | string | 柚子AI的系统提示词（控制对话风格和行为规则） |
| `initialAffection` | number | 初始好感度（默认30） |
| `maxAffection` | number | 好感度上限（默认100） |

### 好感度行为对照

| 好感度范围 | 柚子行为 |
|---|---|
| < 40 | 说话谨慎，保留关键信息，偶尔显示距离感 |
| 40-70 | 主动协助，分享观察和推论，关心老师理智状态 |
| > 70 | 主动分享深层线索，对老师安危表现出超越任务的关心 |

**调整建议：**
- 提高 `initialAffection`：让柚子一开始更亲近
- 降低 `initialAffection`：增加前期建立关系的难度
- 修改 `systemPrompt` 中的好感度行为准则来调整各阶段的表现

## 8. endings.json — 结局规则

### 文件结构

数组，每个元素代表一个结局判定规则。按 `priority` 从小到大判定，首个满足条件的结局生效。

### 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `type` | string | 结局类型标识 |
| `description` | string | 结局描述（策划备注） |
| `priority` | number | 判定优先级（1最高，先判定） |
| `conditions` | array | 触发条件数组 |
| `logic` | string | 条件逻辑：`"AND"` 或 `"OR"` |
| `narrativeHint` | string | 叙事提示（供AI生成结局文本） |

### 结局类型

| type | 说明 |
|---|---|
| `FAIL` | 失败结局（理智归零/回合耗尽且分数不足） |
| `HIDDEN` | 隐藏结局（特殊条件触发） |
| `SECRET` | 秘密结局（好感度相关） |
| `PERFECT` | 完美结局（揭露度满格） |
| `FINAL` | 普通结局（揭露度达标但未满格） |

### 条件字段语法

| field | 可用值 | 说明 |
|---|---|---|
| `player.sanity` | 0-100 | 玩家理智值 |
| `player.revelation` | 0-100 | 揭露度 |
| `player.affection` | 0-100 | 好感度 |
| `currentChapter` | `"ch1"`-`"ch5"` | 当前章节 |
| `turn` | 0-100 | 当前回合 |
| `score` | number | 综合分数 |

| operator | 说明 |
|---|---|
| `eq` | 等于 |
| `neq` | 不等于 |
| `lt` | 小于 |
| `lte` | 小于等于 |
| `gt` | 大于 |
| `gte` | 大于等于 |

> 数值类型支持全部6种运算符；字符串类型仅支持 `eq` 和 `neq`。

**修改示例 — 添加新结局：**

```json
{
  "type": "HIDDEN",
  "description": "全物品收集结局",
  "priority": 2,
  "conditions": [
    {"field": "currentChapter", "operator": "eq", "value": "ch5"},
    {"field": "player.sanity", "operator": "gte", "value": 50}
  ],
  "logic": "AND",
  "narrativeHint": "你收集了千禧塔中所有的碎片，拼凑出了完整的图景。真相不在终点——在路上。"
}
```

> ⚠️ 优先级数字越小越先判定。确保 FAIL 类型结局优先级最高，PERFECT 优先级高于 FINAL。

## 12. 修改注意事项

### 通用规则

1. **JSON格式**：所有配置文件必须是合法的JSON格式。修改前建议使用JSON校验工具验证。
2. **ID引用一致性**：地图、谜题、NPC、物品之间的ID引用必须保持一致。例如：
   - `maps.json` 中的 `puzzles` 必须在 `puzzles.json` 中存在
   - `maps.json` 中的 `npcIds` 必须在 `npcs.json` 中存在
   - `maps.json` 中的 `items` 必须在 `items.json` 中存在
   - `puzzles.json` 中的 `requiredItemId` 必须在 `items.json` 中存在
   - `maps.json` 中的 `nextMapId` 必须在 `maps.json` 中存在
3. **修改后生效**：配置文件修改后可通过以下方式生效：
   - **管理端热重载**：修改文件后调用管理端 API `POST /api/admin/data/reload`，无需重启
   - **重启应用**：重启 Spring Boot 应用，配置文件在启动时加载
4. **编码格式**：所有文件使用 UTF-8 编码。

### 数值平衡建议

| 想要的效果 | 调整方向 |
|---|---|
| 游戏更难（更容易失败） | 降低 `maxTurns`、提高 `sanityDecayCurve`、降低 `puzzleRewards`、提高 `failSanityPenalty` |
| 游戏更简单 | 提高 `maxTurns`、降低 `sanityDecayCurve`、提高 `puzzleRewards`、提高 `npcUnlockSanityReward` |
| 更快节奏 | 降低 `stageReportInterval`、降低 `fallbackPuzzleActivationTurns`、降低 `puzzleMinRoundsBase` |
| 更慢节奏（更多探索） | 提高 `stageReportInterval`、提高 `fallbackPuzzleActivationTurns`、提高 `puzzleMinRoundsBase` |
| 更容易达成完美结局 | 降低 PERFECT 结局的 `revelation` 阈值、提高 `explorationRevelationBonus` |
| 更容易触发隐藏结局 | 降低 SECRET 结局的 `affection` 阈值、提高 `initialAffection` |

### 添加新内容的完整流程

**添加新章节：**
1. `story.json` → 添加章节定义
2. `maps.json` → 添加对应地图
3. `puzzles.json` → 添加地图中的谜题
4. `npcs.json` → 添加地图中的NPC
5. `items.json` → 添加地图中的物品
6. `maps.json` → 更新上一张地图的 `nextMapId`
7. `endings.json` → 如需新结局，添加判定规则
8. `game_config.json` → 更新 `chapterRevelationRewards`

**添加新谜题：**
1. `puzzles.json` → 添加谜题定义
2. `maps.json` → 将谜题ID添加到对应地图的 `puzzles` 数组
3. `items.json` → 如需前置物品，添加物品定义
4. `maps.json` → 将前置物品ID添加到对应地图的 `items` 数组

**添加新NPC：**
1. `npcs.json` → 添加NPC定义
2. `maps.json` → 将NPC ID添加到对应地图的 `npcIds` 数组
3. NPC赠礼道具为AI生成道具（显示在线索栏），无需在 `items.json` 中定义。ID格式：`item_{npcId}_gift`

### 常见问题

**Q：修改配置后没有生效？**
A：需要通过管理端 API `POST /api/admin/data/reload` 热重载配置，或重启 Spring Boot 应用。直接修改 JSON 文件不会自动生效。

**Q：新增的物品没有出现在游戏中？**
A：检查物品ID是否已添加到对应地图的 `items` 数组中。物品只有在其所属地图的 `items` 列表中才会被发现。

**Q：NPC没有出现？**
A：检查NPC的 `appearCondition` 是否满足，以及NPC ID是否在当前地图的 `npcIds` 列表中。

**Q：谜题无法激活？**
A：检查谜题ID是否在当前地图的 `puzzles` 列表中，以及 `requiredItemId` 对应的物品玩家是否已持有。

**Q：地图无法切换？**
A：检查当前地图的 `unlockCondition` 是否已满足，以及 `nextMapId` 指向的地图是否存在。

**Q：结局没有触发？**
A：检查 `endings.json` 中的条件是否满足，以及 `priority` 排序是否正确。系统按优先级从高到低判定，首个满足条件的结局生效。
